# Launch Plan

## Recommended Sequence

1. Publish the toolkit as a GitHub repo.
2. Create a simple personal landing page that links to the repo and ZIP download.
3. Publish a short developer-facing article on Hashnode or DEV.
4. Adapt the article for specific ecosystems, including Salesforce, after the language is polished.
5. Use LinkedIn last, with a sanitized post that points to the public framework.

## Why This Sequence

GitHub establishes credibility with technical audiences and vendor engineering teams. A personal landing page gives the offer a stable home. Developer platforms allow discovery outside the employer network. LinkedIn can amplify later without being the only public source.

## Public-Safety Rules

- Do not mention private employer vendor conversations.
- Do not imply any nonpublic relationship among vendors and employers.
- Do not use internal project names, issue keys, screenshots, architecture, or examples.
- Do not describe internal release pipelines as facts.
- Keep examples fictional and generic.
- Use the disclaimer in the README and landing page.
